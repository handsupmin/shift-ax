# Engineering-Discipline Playbook

## Check
- The implementation plan states the required engineering method (for example TDD) and architecture guardrails.
- Bug, CI, and review-failure work follows reproduce-first and stop-the-line discipline.
- Anti-rationalization guardrails stay explicit.

## Approve when
- The plan and execution evidence show the expected engineering method.
- Failure handling recorded reproduction before the fix path moved forward.

## Request changes when
- The plan hand-waves test strategy or architecture boundaries.
- A failure was patched without a clear reproduction or reviewable evidence trail.
