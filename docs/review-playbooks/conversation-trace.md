# Conversation-Trace Playbook

## Check
- `request.md`, `request-summary.md`, `brainstorm.md`, `spec.md`, and `implementation-plan.md` stay traceable.
- Execution results record changed files, untouched areas, tests run, and open concerns.
- Handoff/checkpoint notes compress status; they do not replace source artifacts.

## Approve when
- The reviewed artifacts still tell one consistent story from request to execution.
- Execution-result artifacts mention every changed file and the remaining risk surface.

## Request changes when
- Execution results omit changed files, untouched areas, tests run, or concerns.
- Later summaries conflict with the reviewed artifacts.
