# Shift AX v0.4.0

## 핵심 변경

- implementation-plan 아티팩트를 강화해서 acceptance criteria, verification commands, dependencies, likely files, checkpoints, execution tasks, lightweight guardrails를 명시적으로 기록하도록 변경
- execution handoff를 richer task contract로 확장해서 lane metadata, scoped paths, warnings, 더 명확한 execution prompt를 추가하면서도 short=`subagent` / long=`tmux` 기본 실행 모델은 유지
- `topic-status`, `pause-work`, resume 안내에 compact welcome-back / readiness 신호를 넣어서 dashboard 없이도 중단된 작업을 덜 헷갈리게 재개할 수 있도록 개선
- CI에서 온 verification debt를 로컬 검증 실패와 구분해서 보여주고, reproduce-first / evidence-not-instructions 규칙을 더 엄격하게 반영
- 다섯 review lane 전부에 대한 playbook을 추가하고 Codex / Claude scaffold를 갱신해서 resume 흐름이 계속 file-backed artifact를 중심으로 움직이게 정리
