# Shift AX v0.2.0

**한국어 버전:** [v0.2.0.ko.md](./v0.2.0.ko.md)

## Summary

Shift AX v0.2.0 turns the project from a workflow-heavy source checkout into a more installable product shell.

This release focuses on:

- easier installation
- easier first-run onboarding
- better platform-native entrypoints
- keeping the same docs-first request-to-commit core underneath

## Included in v0.2.0

- conversational shell entrypoints:
  - `ax --codex`
  - `ax --claude-code`
  - `shift-ax --codex`
  - `shift-ax --claude-code`
- automatic first-run onboarding when shared context artifacts are missing
- language selection at first launch:
  - English
  - Korean
- guided onboarding that asks for:
  - business context
  - primary workflow
  - no-guessing policy areas
  - risky domains
  - architecture summary
  - important paths
  - internal terms / aliases
  - default verification commands
- `.ax/project-settings.json` for locale and preferred platform
- product-shell command guidance for slash / dollar command use inside Codex and Claude Code sessions
- npm packaging for public installation
- one-command global installer script

## Verification highlights

- full automated test and build verification
- shell launcher tests for:
  - Codex shell
  - Claude Code shell
  - first-run language + onboarding flow
- packed-tarball global install smoke test
- npm publish completed for:
  - `@handsupmin/shift-ax@0.2.0`

## What stayed the same

The core request-to-commit control plane is unchanged in principle:

- docs-first context resolution
- planning interview
- human plan review
- policy sync before implementation
- structured review lanes
- Lore commit finalization

## Operator takeaway

v0.2.0 is the version where Shift AX starts feeling like a product shell rather than an internal CLI wrapper.

If a team wants:

- a platform-native entrypoint
- automatic first-run setup
- a safer onboarding experience

then v0.2.0 is the recommended baseline.
