# Shift AX v0.2.1

**한국어 버전:** [v0.2.1.ko.md](./v0.2.1.ko.md)

## Summary

Shift AX v0.2.1 is a patch release focused on fixing the first-run shell experience.

The product-shell entrypoint introduced in v0.2.0 worked, but the onboarding dialogue could still be owned by the wrapper before Codex or Claude Code fully took over the terminal. In real use, that could lead to swallowed input, overwritten prompts, and a generally broken first-run experience.

v0.2.1 moves that onboarding into the platform session itself for normal shell usage.

## Included in v0.2.1

- first-run shell onboarding now happens inside the active Codex or Claude Code session
- the wrapper no longer owns the normal first-run onboarding conversation before platform handoff
- explicit automation paths still remain available:
  - `ax onboard-context --input ...`
  - `ax onboard-context --discover ...`
- shell settings persistence still works for locale and preferred platform

## Why this release matters

This release fixes the highest-friction issue in the v0.2.0 product-shell flow:

- terminal ownership is no longer split across two UIs during normal first-run shell use
- the onboarding conversation now happens where the user actually intends to work
- Codex / Claude shell entry is more consistent with the “talk in one place” product goal

## Verification highlights

- targeted shell/onboarding tests
- full automated test suite
- full build verification
- real wrapper handoff confirmation for Codex shell launch

## Operator takeaway

If you are using the product-shell entrypoints:

- `ax --codex`
- `ax --claude-code`

then v0.2.1 is the recommended version over v0.2.0.
