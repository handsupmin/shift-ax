# Shift AX v0.2.1

**English version:** [v0.2.1.md](./v0.2.1.md)

## 요약

Shift AX v0.2.1은 **first-run shell 경험 수정**에 집중한 패치 릴리즈입니다.

v0.2.0에서 product-shell entrypoint 자체는 도입됐지만, 온보딩 대화가 여전히 wrapper 쪽에서 먼저 터미널을 잡고 진행되는 경로가 남아 있었습니다. 실제 사용에서는 이것이 입력 씹힘, 프롬프트 덮어쓰기, 어색한 first-run UX로 이어질 수 있었습니다.

v0.2.1은 정상 first-run shell 사용에서 그 온보딩을 플랫폼 세션 내부로 옮깁니다.

## v0.2.1에 포함된 것

- first-run shell onboarding이 이제 Codex / Claude Code 세션 안에서 진행됨
- wrapper가 정상 first-run onboarding 대화를 먼저 소유하지 않음
- 명시적 자동화 경로는 그대로 유지:
  - `ax onboard-context --input ...`
  - `ax onboard-context --discover ...`
- locale / preferred platform 저장도 계속 동작

## 왜 중요한가

이 릴리즈는 v0.2.0 product-shell 흐름에서 가장 마찰이 컸던 지점을 고칩니다.

- 정상 first-run shell 사용에서 더 이상 두 UI가 같은 터미널을 번갈아 잡지 않음
- 온보딩 대화가 사용자가 실제로 일하려는 플랫폼 세션 안에서 진행됨
- Codex / Claude shell 진입 경험이 “한 곳에서 대화한다”는 제품 목표에 더 맞아짐

## 검증 포인트

- shell/onboarding targeted tests
- 전체 자동 test suite
- 전체 build 검증
- Codex shell launch 기준 실제 wrapper handoff 확인

## 운영자 관점 결론

아래 product-shell entrypoint를 실제로 쓴다면:

- `ax --codex`
- `ax --claude-code`

v0.2.0보다 **v0.2.1을 쓰는 것이 맞습니다.**
